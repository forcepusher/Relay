# WebSocketRelay
Networking as simple as it gets.  
  
Key priorities:  
1. Debugability - Readable Json data stream for developing, binary for shipping.
2. Protability - Relay server runtime ships with the package, double-click-ready.
3. Testability - Integration/unit tests using the portable runtime for quick QA.
  
Yes, it actually serializes your state to Json so you can easily spot bugs.  
That is the core difference for improving your debugging experience.
